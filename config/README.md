# DocAI Configuration

## Configuration Files

### `docai_config.json.template`
- **Template file** - tracked in git
- Contains default configuration values
- **DO NOT edit this file** - it's the template for new installations

### `docai_config.json`
- **User-specific config** - NOT tracked in git (in `.gitignore`)
- Auto-generated from template on first use
- Contains your personal settings:
  - Selected model
  - Execution mode (python/docker)
  - Documentation sources
  - Timestamps and user data

## How It Works

When you first run DocAI:

1. **ConfigManager** checks if `config/docai_config.json` exists
2. If NOT found, it copies `docai_config.json.template` → `docai_config.json`
3. Your changes are saved to `docai_config.json`
4. The file is NOT tracked in git, so no conflicts between users

## For Developers

### Adding New Configuration Options

1. Edit `docai_config.json.template` with your new defaults
2. Commit the template to git
3. Users will get the new fields automatically on next pull (their existing config remains unchanged)

### Resetting Configuration

To reset to defaults:
```bash
rm config/docai_config.json
# Next run will recreate from template
```
